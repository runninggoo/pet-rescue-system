## 一、截图规范

### 1.1 命名规则

```
SC-{章节}{序号:02d}-{页面/模块}-{功能描述}.png

示例：
SC-0301-数据库-ER图总览.png    # 第3章第1张：数据库ER图
SC-0401-登录-多角色切换.png    # 第4章第1张：登录页多角色切换
SC-0408-统计-9图合一展示.png   # 第4章第8张：统计页面
```

**命名解析**：
- `SC`：Screenshot 缩写
- `{章节}`：论文章节号（如 03=第3章，04=第4章）
- `{序号}`：该章节内的顺序号（01~99）
- `{页面/模块}`：所属模块（登录、宠物列表、数据库等）
- `{功能描述}`：具体展示的功能点

```

```

### 1.3 截图要求

| 要求项 | 标准 |
|--------|------|
| 格式 | PNG，建议 1920×1080 或全窗口截图 |
| 浏览器 | Chrome/Edge（登录态需先完成登录）|
| 隐藏内容 | 尽量隐藏浏览器地址栏和标签栏 |
| 标注 | 仅标注1-2个关键亮点，用红色矩形框 |
| 文字说明 | 每个截图附1-2句简短说明 |

### 1.4 状态说明

| 状态 | 含义 |
|------|------|
| ⏳ 待采集 | 尚未截图 |
| 📷 已采集 | 原始截图已完成，未标注 |
| 🎯 已标注 | 已用红框标注亮点，可提交论文 |
| ✅ 已提交 | 已放入论文文档 |

## 五、标注规范

### 5.1 标注工具推荐

- **FastStone Capture**：轻量截图 + 标注（推荐）

- **微信/QQ 截图工具**：基础标注

- 红框线宽：2px
- 红框颜色：`#E74C3C`（亮红色）
