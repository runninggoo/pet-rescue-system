## 📌 一、项目基本信息（论文摘要用）

### 1.1 项目名称
**基于Spring Boot的宠物救助收养系统**

### 1.2 项目背景
随着社会经济与宠物饲养文化的发展，流浪宠物数量激增，已成为不容忽视的城市治理与公共卫生问题。当前宠物救助收养领域面临三大核心痛点：
1. **信息碎片化**导致匹配效率低下
2. **流程非标准化**引发运营与信任危机
3. **多主体协同缺失**造成资源浪费

### 1.3 核心技术栈
| 类别 | 技术 |
|------|------|
| 后端框架 | Spring Boot 2.7.0 + Spring Security |
| 持久层 | MyBatis Plus 3.5.3.1 |
| 数据库 | MySQL 8.0+ |
| 前端框架 | Vue.js 2.6.14 + Element UI 2.15.13 |
| 认证方式 | JWT (JSON Web Token) + Refresh Token 双Token架构 |
| 构建工具 | Vite（前端开发服务器） |

### 1.4 系统功能模块
- ✅ 用户认证与权限管理（5种角色：admin/institution_admin/volunteer/adopter/pet_hospital）
- ✅ 宠物信息管理（CRUD + 多条件搜索 + 品种两级联动分类）
- ✅ 领养申请流程（提交 → 审核 → 签约 → 回访）**【v1.1.3 新增回访记录】**
- ✅ 救助站智能推荐（基于Haversine距离算法 + 多因素权重评分）
- ✅ 数据统计看板（9个ECharts图表 + 简化版统计页）**【v1.1.3 新增 simple-stats.html】**
- ✅ 领养申请管理页面（管理员审核申请 + 6种状态流转）
- ✅ 健康档案管理（疫苗/体检/治疗时间线记录）
- ✅ 协同任务管理（志愿者任务接单 + 管理员发布）
- ✅ **志愿者成长体系（积分/签到/等级/排行榜）**【v1.1.5 新增】
- ✅ **宠物收藏功能（收藏/取消收藏/我的收藏）**【v1.2.0 新增】
- ✅ **宠物医院入口页面**【v1.1.3 新增】
- ✅ **帮助中心页面**【v1.1.3 新增】

### 1.5 创新点
1. **全链路数字化** - 覆盖救助到跟踪的完整业务流程
2. **多角色协同** - 支持5类用户角色的精细权限控制
3. **救助站智能推荐** - 基于地理位置和容量的智能匹配算法
4. **数据驱动决策** - 集成统计看板支持管理决策

## 📌 四、关键代码片段（论文第四章用）

### 4.1 Haversine距离计算算法

```java
/**
 * 使用Haversine公式计算两点之间的距离
 * @param lat1 用户纬度
 * @param lon1 用户经度
 * @param lat2 救助站纬度
 * @param lon2 救助站经度
 * @return 距离（公里）
 */
public static double calculateDistance(double lat1, double lon1, double lat2, double lon2) {
    final double R = 6371; // 地球半径（公里）
    double latDistance = Math.toRadians(lat2 - lat1);
    double lonDistance = Math.toRadians(lon2 - lon1);
    double a = Math.sin(latDistance / 2) * Math.sin(latDistance / 2)
            + Math.cos(Math.toRadians(lat1)) * Math.cos(Math.toRadians(lat2))
            * Math.sin(lonDistance / 2) * Math.sin(lonDistance / 2);
    double c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
    return R * c;
}
```

### 4.4 智能推荐权重评分算法

```java
/**
 * 计算救助站综合评分
 * @param distance 距离（公里）
 * @param maxDistance 最大搜索距离
 * @param remainingCapacity 剩余容量
 * @param maxCapacity 最大容量
 * @param medicalLevel 医疗等级（A/B/C）
 * @return 综合评分（0-100）
 */
public int calculateScore(double distance, double maxDistance,
                          int remainingCapacity, int maxCapacity, String medicalLevel) {
    // 距离得分：距离越近得分越高（权重40%）
    double distanceScore = 40 * (1 - distance / maxDistance);

    // 容量得分：剩余容量比例越高得分越高（权重30%）
    double capacityRatio = (double) remainingCapacity / maxCapacity;
    double capacityScore = 30 * capacityRatio;

    // 医疗等级得分（权重30%）
    double medicalScore = 30;
    if ("B".equals(medicalLevel)) medicalScore = 20;
    else if ("C".equals(medicalLevel)) medicalScore = 10;

    return (int) (distanceScore + capacityScore + medicalScore);
}
```

## 📌 十一、答辩可能问题及回答

### Q1: 为什么选择Spring Boot框架？
**A**: Spring Boot简化了Spring应用的初始搭建和开发过程，提供了自动配置功能，能够快速构建独立运行的生产级Spring应用程序。

### Q2: 如何保证系统安全性？
**A**: 本系统采用JWT（JSON Web Token）进行用户认证，结合Spring Security实现基于角色的权限控制。同时引入Refresh Token双Token机制，Access Token短期有效（30分钟），Refresh Token存数据库（7天），支持主动注销和安全刷新。

### Q3: 智能推荐算法如何实现？
**A**: 采用Haversine公式计算用户与救助站之间的地理距离，结合救助站容量和医疗等级进行加权评分（距离40% + 容量30% + 医疗等级30%），返回综合评分最高的救助站列表。

### Q4: 数据库设计遵循什么原则？
**A**: 遵循数据库设计三范式，确保数据完整性，避免数据冗余。核心表之间通过外键关联，保证数据一致性。使用MyBatis Plus的逻辑删除（@TableLogic）实现软删除。

### Q5: 系统有什么创新点？
**A**: 1. 全链路数字化管理；2. 多角色协同工作；3. 基于地理位置的救助站智能推荐；4. 完整的领养流程追踪；5. 志愿者成长激励体系；6. 个性化宠物收藏。

### Q6: JWT双Token机制相比单Token有什么优势？
**A**: Access Token频繁传输存在泄露风险，双Token机制将短期认证令牌与长期刷新令牌分离，Access Token泄露后攻击窗口仅30分钟；Refresh Token存服务端数据库，支持主动注销任意设备会话；结合User-Agent和IP记录，可追踪令牌创建时的设备信息。

### Q7: 志愿者成长体系如何设计？
**A**: 6级成长体系（新手上路 → 公益之星），积分来源包括每日签到（+5分/天）、连续7天签到额外奖励（+20分）、完成任务（+10分/任务）。系统自动根据累计积分计算当前等级，排行榜展示TOP10志愿者。

### Q8: 宠物收藏功能如何实现？
**A**: 用户可对心仪宠物点击爱心图标收藏，数据存储在 pet_favorite 表（user_id + pet_id，唯一约束防止重复收藏）。个人中心「我的收藏」Tab展示收藏列表，支持取消收藏。
