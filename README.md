# 🐾 宠物救助收养系统

## 📋 项目简介

宠物救助收养系统是一个基于Spring Boot + Vue.js的前后端分离的Web应用，旨在解决流浪宠物救助收养领域的数字化管理问题。系统覆盖从"救助发现→领养审核→健康管理→后续跟踪"的全业务流程，支持多角色（救助机构、领养人、志愿者、宠物医院、管理员）在线协同工作。

## 🎯 项目背景

随着社会经济与宠物饲养文化的发展，流浪宠物数量激增，已成为不容忽视的城市治理与公共卫生问题。当前宠物救助收养领域面临三大核心痛点：

1. **信息碎片化**导致匹配效率低下
2. **流程非标准化**引发运营与信任危机
3. **多主体协同缺失**造成资源浪费

本系统通过数字化手段，为各参与方提供协作工具，建立标准化、可追溯的收养管理体系。

## 🚀 快速开始

### 环境要求
- JDK 1.8+
- MySQL 5.7+
- Node.js（前端开发）

### 数据库配置
```sql
CREATE DATABASE pet_rescue CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE pet_rescue;
-- 执行 sql/schema.sql 创建表结构
-- 执行 sql/test_data.sql 插入测试数据
```

### 后端启动
1. 复制 `backend/src/main/resources/application-example.yml` 为 `application.yml`，填入数据库密码和 JWT 密钥
2. 使用 IntelliJ IDEA 打开 `backend` 目录，等待 Maven 依赖下载
3. 运行 `PetRescueSystemApplication.main()`
4. 默认端口：**8081**，接口根路径：`/api`

### 前端启动
1. 进入 `frontend` 目录，执行 `npm install`
2. 执行 `npm run dev` 启动开发服务器
3. 访问提示的本地地址（默认 http://localhost:5173）

> 也可直接双击 `frontend/public/login.html` 打开静态页面（需后端运行在 8081 端口）

## 🎯 项目特色

### 创新点
1. **全链路数字化** - 覆盖救助到跟踪的完整流程
2. **多角色协同** - 支持5类用户角色的精细权限控制
3. **救助站智能推荐** - 基于Haversine地理距离的权重评分算法
4. **个性化宠物推荐** - 基于用户行为记录的偏好向量推荐
5. **数据驱动决策** - 集成统计看板支持管理决策

### 技术亮点
1. **前后端分离架构** - Vue.js + Spring Boot
2. **安全的认证机制** - JWT + Spring Security
3. **高性能数据库设计** - 规范的表结构 + 索引优化
4. **响应式前端** - 适配PC和移动端
