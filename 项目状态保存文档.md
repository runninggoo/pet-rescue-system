## 2026-04-12（本次对话）健康档案页面 DOM 结构修复

### 问题现象

访问 `health-record.html` 后，页面显示 Vue 表达式文字（如 `{{editForm.id}}`、`{{getRoleNameH(currentRole)}}`），页面功能完全失效。

### 根本原因

`health-record.html` 行 846-848 区域多出了两个多余的 `</div>` 闭合标签：

```html
<span :style="..."> {{ getRoleNameH(currentRole) }}
</span>    ← 缩进错误，多余的 </span>
</div>     ← 多余的 </div>（hero-right 已闭合）
</div>     ← 多余的 </div>（hero-banner 已闭合）
```

这导致 Vue 模板的 DOM 层级结构被破坏，Vue 2 无法正确解析模板，所有表达式直接暴露为静态文本。

### 修复方案

删除行 846-848 多余的 `</span>` 和两个 `</div>`，恢复正确的 DOM 结构：

```html
<span :style="..."> {{ getRoleNameH(currentRole) }} </span>
```

### 修改文件

- `backend/src/main/resources/static/health-record.html`
- `frontend/public/health-record.html`（同步）

### 浏览器验证

- Vite (5173)：`http://localhost:5173/health-record.html` ✅ 正常渲染
- 后端 (8081)：`http://localhost:8081/api/health-record.html` ✅ 正常渲染

### 预防经验

**Vue 表达式 `{{...}}` 直接显示的常见原因**：
1. `</div><!-- end #app -->` 提前闭合，导致对话框移到 `#app` 外部
2. DOM 标签未匹配闭合（如本例多余 `</div>`）
3. HTML 结构被截断或嵌套错误

诊断方法：检查 `id="app"` 内部所有标签的闭合是否完整匹配。

## 2026-04-03 下午 品种筛选 Bug 完全修复

### 问题描述

宠物列表页面「宠物类型 + 品种」两级联动筛选有严重错误：
- 选「猫」后，品种下拉显示「无数据」
- 选「狗」后，品种下拉显示：狸花猫、无毛猫、泰迪犬、哈士奇、金毛寻回犬、拉布拉多、泰迪犬、哈士奇、柯基、柴犬
- 选「小动物」后，品种下拉显示：萨摩耶、中华田园犬、边境牧羊犬、荷兰猪

### 根本原因（两层 Bug 叠加）

#### Bug 1：后端 `getChildren` 方法误用 `isNull`（核心原因）

`PetCategoryServiceImpl.getChildren(Long parentId)` 中写的是：

```java
wrapper.isNull(PetCategory::getParentId); // ❌ 错：查所有 parent_id 为 NULL 的（根节点）
wrapper.eq(PetCategory::getIsActive, 1);
wrapper.orderByAsc(PetCategory::getSort);
```

正确应为 `isNotNull(PetCategory::getParentId)`（查所有子品种）。

同时 `getRootCategories` 方法中，填充 children 时对每个 root 都调用 `getChildren(id)`，由于方法本身有 bug，导致所有大类的子品种全部混在一起。

#### Bug 2：前端 `handleSearch()` 清空品种列表

```javascript
handleSearch() {
    this.breedChildren = [];  // ❌ 每次搜索都清空，品种下拉变空
    this.currentPage = 1;
    this.loadPets();
}
```

每次搜索后品种下拉就空了，用户误以为「没有品种可选」。

#### Bug 3：`/api/category/**` 未加入白名单

`SecurityConfig.java` 未将 `/api/category/**` 设为 `permitAll`，导致品种列表 API 需要认证（403），进一步加剧了数据错乱。

### 修复方案

#### Step 1 - `PetCategoryServiceImpl.getRootCategories()` 批量填充 children

不再对每个 root 调用 N+1 次 `getChildren()`，改为**一次性查出所有子品种按 parentId 分组**，再按 parentId 正确挂载到各自根分类下：

```java
// 一次性查出所有子品种（parent_id IS NOT NULL），按 parentId 分组
LambdaQueryWrapper<PetCategory> childWrapper = new LambdaQueryWrapper<>();
childWrapper.isNotNull(PetCategory::getParentId)
        .eq(PetCategory::getIsActive, 1)
        .orderByAsc(PetCategory::getSort);
List<PetCategory> allChildren = baseMapper.selectList(childWrapper);

// 按 parentId 分组
java.util.Map<Long, List<PetCategory>> childMap = new java.util.HashMap<>();
for (PetCategory child : allChildren) {
    childMap.computeIfAbsent(child.getParentId(), k -> new ArrayList<>()).add(child);
}

// 挂载到各根分类下
for (PetCategory root : roots) {
    root.setChildren(childMap.getOrDefault(root.getId(), new ArrayList<>()));
}
```

#### Step 2 - `handleSearch()` 移除清空品种列表

```javascript
handleSearch() {
    // 不再清空 breedChildren，避免品种下拉每次搜索后消失
    this.currentPage = 1;
    this.loadPets();
}
```

#### Step 3 - 新增 `loadBreedsByCategory(type)` 前端方法

`loadCategories()` 只负责拉所有大类（大类下拉），品种列表通过 `/api/category/list?type=X` 精准获取：

```javascript
loadBreedsByCategory(type) {
    if (!type) { this.breedChildren = []; return Promise.resolve(); }
    return axios.get('/api/category/list', { params: { type: type } })
        .then(res => {
            if (res.data.code === 200) {
                const cats = res.data.data.categories || [];
                this.breedChildren = (cats[0] && cats[0].children) ? cats[0].children : [];
            }
        });
},

onCategoryTypeChange(type) {
    this.filters.breed = '';
    if (!type) {
        this.breedChildren = [];
        this.handleSearch();
        return;
    }
    // 精准拉该大类的品种，再搜索
```

## 2026-04-03 志愿者成长体系开发

### 本次开发概述

**主题**：志愿者积分成长体系（游戏化激励机制）

灵感来源：借鉴 Bilibili 用户等级体系，参考毕设实际需求定制

### 需求分析

原始问题：志愿者任务的 `rewardPoints` 字段有积分值，但无用户积分余额、无等级概念、无积分历史记录、无积分获取/消费场景、无个人中心积分展示。积分形同虚设。

采用方案 B（轻量整合）：volunteer_profile + point_history + level_rule 三张表，有历史可追溯、有等级成就感。

### 数据库设计（3张表）

```sql
CREATE TABLE volunteer_profile (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL UNIQUE,
    total_points INT DEFAULT 0,      -- 累计积分
    available_points INT DEFAULT 0,   -- 可用积分
    current_level INT DEFAULT 1,     -- 当前等级 1-6
    current_level_name VARCHAR(20),    -- 等级名称
    total_tasks INT DEFAULT 0,        -- 累计完成任务数
    total_hours DECIMAL(8,2),        -- 累计服务时长
    title VARCHAR(50),                -- 专属称号
    continuous_sign_days INT DEFAULT 0, -- 连续签到天数
    last_sign_date DATE,             -- 最后签到日期
    created_at, updated_at, deleted
);

CREATE TABLE point_history (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    points INT NOT NULL,             -- 正数=获得，负数=消费
    balance INT NOT NULL,             -- 变动后余额
    source VARCHAR(30) NOT NULL,     -- TASK_COMPLETE/DAILY_SIGN/CONTINUOUS_SIGN_7...
    related_id BIGINT,               -- 关联ID
    description VARCHAR(200),
    created_at
);

CREATE TABLE level_rule (
    id, level, level_name, min_points, max_points,
    badge_icon, privilege_desc, level_color, sort_order
);
```

### API 设计（6个端点）

```
GET  /api/volunteer/profile       — 成长档案（含等级进度）

GET  /api/volunteer/leaderboard  — 排行榜 TOP10
POST /api/volunteer/daily-sign   — 每日签到
GET  /api/volunteer/level-rules  — 等级规则
PUT  /api/volunteer/title        — 更新称号
```

## 2026-04-02 下午 品种管理弹窗 Bug 完全修复 ✅

### 根本原因

Element UI 的 `el-dialog` 在渲染时会生成 `el-dialog__wrapper` 层并挂载到 `document.body`，同时创建独立的 Vue 组件实例上下文。这导致 Dialog 内的 `@click` 事件回调无法访问根 `#app` Vue 实例的方法，所有按钮点击完全失效。

### 修复方案

将品种管理弹窗从 `el-dialog` 替换为**原生 div 覆盖层**，彻底绕过 Vue 隔离问题。

### 一、宠物列表分页 Bug 完全修复 ✅

#### 根本原因（三层叠加）

1. 数据库 pet 表的 `deleted` 列为 NULL 而非 0 — MyBatis Plus 全局逻辑删除 COUNT 查询不匹配 NULL 值

1. 数据库：`UPDATE SET deleted=0 WHERE deleted IS NULL`
2. 配置文件：移除 `application.yml` 全局逻辑删除配置
3. 后端分页逻辑：放弃 MyBatis Plus 自动 COUNT，改为手动 `selectCount` + Java 层分页截取
