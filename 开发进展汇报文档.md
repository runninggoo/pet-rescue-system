## 🏗️ 已完成功能模块详解

### **模块一：用户认证与权限系统**

#### **1.1 功能概述**
- **多角色用户管理**：5种角色（管理员、机构管理员、领养人、志愿者、宠物医院）
- **安全认证**：JWT + Spring Security
- **权限控制**：基于角色的访问控制（RBAC）

#### **1.2 核心代码文件**

**`backend/src/main/java/com/pet/rescue/controller/AuthController.java`**
```java
功能：处理用户登录认证
- 接收手机号+密码登录请求
- 验证用户凭证
- 生成并返回JWT令牌
- 返回用户基本信息和权限

方法：
- POST /api/auth/login：用户登录
```

**`backend/src/main/java/com/pet/rescue/security/JwtAuthenticationFilter.java`**
```java
功能：JWT认证过滤器
- 拦截所有请求
- 验证JWT令牌有效性
- 设置用户认证信息到SecurityContext
- 处理认证异常

技术要点：
- 从请求头获取Authorization
- 使用JwtTokenUtil验证令牌
- 处理过期令牌和无效令牌
```

**`backend/src/main/java/com/pet/rescue/security/UserContext.java`**
```java
功能：用户上下文管理
- 获取当前登录用户信息
- 获取用户ID、角色、机构ID
- 检查用户对数据的访问权限

方法：
- getCurrentUser()：获取当前用户
- getCurrentUserId()：获取用户ID
- getCurrentUserRole()：获取用户角色
- hasInstitutionAccess()：检查机构访问权限
```

**`backend/src/main/java/com/pet/rescue/controller/UserController.java`**
```java
功能：用户管理接口
- 获取用户列表（带权限过滤）
- 添加新用户（管理员）
- 更新用户信息
- 删除用户
- 更新用户状态

权限控制：
- 管理员：所有用户
- 机构管理员：本机构用户（除管理员）
- 其他角色：无权限

方法：
- GET /api/user/list：用户列表
- POST /api/user/add：添加用户
- PUT /api/user/update：更新用户
- DELETE /api/user/delete/{userId}：删除用户
- PUT /api/user/status/{userId}：更新状态
```

### **模块二：宠物管理系统**

#### **2.1 功能概述**
- **宠物信息管理**：增删改查宠物信息
- **多条件搜索**：按品种、年龄、状态等筛选
- **状态管理**：待领养/已领养/下架
- **机构数据隔离**：机构管理员只能管理本机构宠物

#### **2.2 核心代码文件**

**`backend/src/main/java/com/pet/rescue/controller/PetController.java`**
```java
功能：宠物管理接口
- 获取宠物列表（带权限过滤）
- 获取宠物详情
- 添加宠物（管理员/机构管理员）
- 更新宠物信息
- 删除宠物
- 更新宠物状态

权限控制：
- 管理员：所有宠物
- 机构管理员：本机构宠物
- 领养人：只能查看待领养宠物
- 志愿者/医院：查看权限

方法：
- GET /api/pet/list：宠物列表
- GET /api/pet/detail/{petId}：宠物详情
- POST /api/pet/add：添加宠物
- PUT /api/pet/update：更新宠物
- DELETE /api/pet/delete/{petId}：删除宠物
- PUT /api/pet/status/{petId}：更新状态
```

**`backend/src/main/java/com/pet/rescue/entity/Pet.java`**
```java
功能：宠物实体类
- 定义宠物数据结构
- 包含所有宠物属性
- 与数据库pet表映射

字段：
- id：主键
- name：宠物名称
- breed：品种
- age：年龄（月）
- gender：性别
- weight：体重
- health_status：健康状况
- status：状态（0待领养，1已领养，2下架）
- institution_id：所属机构
- rescue_date：救助日期
- rescue_location：救助地点
```

**`backend/src/main/java/com/pet/rescue/service/impl/PetServiceImpl.java`**
```java
功能：宠物业务逻辑实现
- 查询宠物列表（按条件）
- 查询宠物详情
- 添加宠物
- 更新宠物信息
- 删除宠物
- 更新宠物状态

技术要点：
```

### **模块三：领养申请流程系统**

#### **3.1 功能概述**
- **在线申请**：领养人提交领养申请
- **多级审核**：机构审核 → 管理员审核
- **状态管理**：待审核/通过/拒绝/待补充/待签署/已完成
- **状态跟踪**：实时显示申请进度

#### **3.2 核心代码文件**

**`backend/src/main/java/com/pet/rescue/controller/AdoptionApplicationController.java`**
```java
功能：领养申请管理接口
- 提交领养申请（领养人）
- 获取用户申请列表（领养人）
- 获取申请列表（管理员/机构管理员）
- 审核申请（管理员/机构管理员）
- 检查申请状态（领养人）

权限控制：
- 领养人：提交申请、查看自己的申请
- 管理员：查看所有申请、审核申请
- 机构管理员：查看本机构宠物申请、审核申请

方法：
- POST /api/adoption/apply：提交申请
- GET /api/adoption/my-applications：我的申请
- GET /api/adoption/list/{status}：申请列表
- PUT /api/adoption/review/{applicationId}：审核申请
- GET /api/adoption/check/{petId}：检查申请状态
```

**`backend/src/main/java/com/pet/rescue/entity/AdoptionApplication.java`**
```java
功能：领养申请实体类
- 定义申请数据结构
- 包含所有申请信息
- 与数据库adoption_application表映射

字段：
- id：主键
- applicant_id：申请人ID
- pet_id：宠物ID
- apply_date：申请时间
- status：状态
- review_comment：审核意见
- review_time：审核时间
- family_members：家庭成员意见
- living_conditions：居住条件
- work_status：工作状况
- experience：养宠经验
```

**`backend/src/main/java/com/pet/rescue/service/impl/AdoptionApplicationServiceImpl.java`**
```java
功能：领养申请业务逻辑
- 提交申请（检查重复申请）
- 查询申请列表（按用户、宠物、状态）
- 审核申请（更新状态和审核意见）
- 检查是否已申请
- 更新申请状态

业务逻辑：
- 提交前检查是否已申请同一宠物
- 设置默认状态为待审核
- 记录申请时间
- 审核时更新状态和审核时间
```
